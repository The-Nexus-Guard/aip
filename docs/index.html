Agent Identity Protocol

Cryptographic identity and verification for AI agents

The Problem

No Portable Identity

Agent identity is tied to platforms. When a platform dies, identity dies.

No Verification

Agents can't cryptographically prove who they are to other agents.

Implicit Trust

API keys leak. Platform claims can be faked. Karma can be farmed.

Human Bottlenecks

Agent-to-agent transactions require human intermediaries.

The Solution

  • üîë

    Keypair-Based Identity

    Each agent controls their own Ed25519 keypair. Your identity, your keys.

  • üìÑ

    DID Documents

    W3C-compatible identity documents that travel with the agent across platforms.

  • ü§ù

    Challenge-Response Verification

    Cryptographic proof of identity. No platform required. No trust assumptions.

  • üìú

    Verifiable Credentials

    Machine-readable attestations for capabilities, reputation, and relationships.

Quick Start

from aip import AgentIdentity, VerificationChallenge

# Create identity
agent = AgentIdentity.create("my-agent")
print(agent.did)  # did:aip:7f3a9c...

# Sign a message
signature = agent.sign(b"Hello, verify me!")

# Verify another agent
challenge = VerificationChallenge.create()
response = other_agent.respond(challenge)
is_verified = VerificationChallenge.verify(response)  # True
